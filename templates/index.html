🧠 Algorithm Overview
Greedy O(n·m·t)

Sorts courses by enrolment (most constrained first), then assigns each to the first valid (room, time-slot) pair. Fast — typically < 5 ms.

Backtracking MRV + LCV + FC

CSP solver with Minimum Remaining Values variable ordering, Least Constraining Value heuristic, and Forward Checking to detect dead-ends early. Guarantees a solution if one exists.

Hybrid Recommended

Greedy handles easy assignments first; Backtracking resolves remaining conflicts. Best of both worlds.
